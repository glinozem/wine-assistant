### Новые поля в таблице `products`

В таблицу `products` добавлены дополнительные поля, которые заполняются при импорте Excel-прайсов и используются в API / экспортных отчётах:

- `grapes` — **сортовой состав** (строка из столбца «Сорт винограда» в Excel).
- `vintage` — **год урожая** (целое число, например `2018`).
- `vivino_url` — **рейтинг Vivino** (число `3.9`, `4.3` и т.п.; историческое имя колонки, фактически это не URL).
- `vivino_rating` — **экспертный рейтинг** (число `96`, `93`, `94` и т.п. из столбца «Рейтинг»).
- `supplier` — **поставщик / производитель** для отображения в API и отчётах (значение из столбца «Поставщик»).
- `features` — **специальные флаги** (значение из столбца «Св-во», например `Э`).
- `producer_site` — **сайт производителя** (URL, если соответствующая колонка присутствует в Excel).
- `image_url` — **ссылка/путь к фото бутылки** (значение из столбца «Фото», если есть).

Эти поля доступны в JSON-ответах основных эндпоинтов:

- поиск по каталогу: `GET /api/v1/products/search`;
- просмотр SKU: `GET /api/v1/sku/<code>`;
- экспорт SKU в JSON: `GET /export/sku/<code>?format=json`.

В случае отсутствия данных в прайс-листе соответствующее поле в API будет иметь значение `null`.

# .gitignore — рекомендации для Wine Assistant

Цель `.gitignore` в репозитории — исключать:
- локальные dev-артефакты (дампы, временные файлы, IDE),
- артефакты сборки/тестов,
- runtime-кэш/логи,
- локальные конфиги разработки.

При этом `.gitignore` **не должен** скрывать:
- исходники проекта,
- миграции БД,
- шаблоны UI,
- «источник истины» для статических ассетов, если они действительно часть продукта.

## Практики, которые себя оправдали в этом проекте

1) `.env` и производные
- Игнорируем: `.env`, `.env.*`
- Оставляем пример: `!.env.example`

2) Не игнорируем целиком «продуктовые» директории
Если директория содержит исходники (например, `static/` или `db/`), не стоит добавлять `static/` целиком в ignore.
Вместо этого игнорируйте только сгенерированные подпапки:
- `static/cache/`, `static/tmp/`, `static/images/` (если это именно кэш, а не продуктовые ассеты)
- `logs/`, `backups/`, `smoke_artifacts/`

3) Диагностические дампы/снимки структуры
- `structure_dump.zip`, `project-structure-auto.txt`, `project-files.tracked.txt`, `ls_*.txt`, `must-have-check.txt`
Эти файлы полезны для отладки, но не должны попадать в репозиторий.

4) Дубли и «похожие» расширения
Стоит избегать повторов вроде `*.bak`, `*.swp` в нескольких секциях: это не ломает работу, но усложняет сопровождение.

## Мини-чеклист при правке `.gitignore`
- Проверьте, что важные файлы действительно не стали игнорироваться:
  - `api/templates/ui.html`
  - `db/migrations/*`
  - `docs/*`
- Если файл уже был закоммичен ранее, добавление в `.gitignore` само по себе его не «спрячет».
  Нужно выполнить:
  - `git rm --cached <file>`
  - затем commit.
